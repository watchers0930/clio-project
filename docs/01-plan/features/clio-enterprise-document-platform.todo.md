# CLIO Enterprise Document Platform TODO

## 목적
- CLIO를 `기업 문서 운영 플랫폼` 방향으로 재정렬한다.
- 핵심 뼈대는 `저장 -> 공유 -> 코멘트 -> 반영 -> 검색 -> 재활용` 흐름으로 잡는다.
- `계약리스크`, `문서생성`, `회의록`, `보고서`, `메모`는 이 뼈대 위에 올라가는 확장 기능으로 재배치한다.

## 제품 정의
- 한 줄 정의:
  - `기업이 생산하고 관리하는 모든 문서를 저장, 공유, 검색, 검토, 업데이트하는 문서 운영 플랫폼`
- 금지 표현:
  - `계약 리스크 플랫폼`
  - `문서 생성 툴`
  - `메신저/캘린더 중심 협업툴`
- 유지할 표현:
  - `문서허브`
  - `문서 운영`
  - `문서 협업`
  - `공유와 반영`
  - `검색과 재활용`

## 현재 진행 상태
- 완료:
  - 로그인, 사이드바, 대시보드, 문서허브, 검색, 문서생성 화면의 상단 메시지를 `기업 문서 운영 플랫폼` 관점으로 1차 통일
  - 로그인 후 첫 진입 경로를 `문서허브(/files)` 기준으로 조정
  - 사이드바를 `문서 운영 / 협업 / 업무 지원 · 관리` 관점으로 재정렬
  - 문서허브, 검색, 문서생성 사이의 이동 흐름을 다시 연결
  - 문서 상세 화면에 `Document Ops` 요약, 운영 상태 카드, 관련 문서 패널을 추가
  - 문서 상세 API에 댓글 수, 공유 링크 수, 버전 수, 관련 문서 목록 메타를 추가
  - 인라인 댓글 패널을 문서 상세의 고정 협업 영역으로 정리
  - 문서 상세에 공유 링크 목록과 최근 운영 타임라인을 추가
  - 문서 상세/버전 패널에서 `새 버전 생성` CTA를 문서 생성 플로우와 연결
  - 댓글 상태 스키마(`미반영 / 보류 / 반영`)와 반영 버전 기록 필드를 추가
  - 댓글 패널에서 상태 배지와 `보류 / 다시 검토` 액션을 추가
  - 버전 패널에서 각 버전에 반영된 댓글 요약을 함께 표시
  - 검색 결과 액션을 `열기 / 공유 / 코멘트 보기(또는 AI 질의) / 새 문서 활용` 기준으로 재정렬
  - 파일 허브 카드와 최근 업로드 패널에 `공유 / 검색 / 후속 문서 / 계약 분석` 추천 액션을 추가
  - 검색 화면에 관련 문서 추천 패널과 최근 검색 흐름 패널을 추가
  - 파일 공유 모달에 `외부 링크 / 사용자 / 부서` 기준 내부 공유 관리를 추가
  - 파일 허브 카드와 최근 업로드 패널에 `코멘트 보기`를 추가하고 연결 문서의 댓글 패널로 이어지게 정리
  - 파일 공유 모달에서 메시지 채널로 바로 `검토 요청`을 보내는 흐름을 추가
  - 문서 공유 모달에서도 메시지 채널로 바로 `검토 요청`을 보내고 메시지 화면에서 문서 카드로 열리게 정리
  - 문서 공유 모달에 사용자/부서 기준 내부 공유를 추가하고 상세 화면에 공유 대상 패널을 노출
  - 문서 목록, 상세, 다운로드, 댓글, 버전, 반영 API에 내부 공유 기반 접근 제어를 연결
  - 문서 상세의 관련 문서 추천 기준을 같은 템플릿/소스 외에 제목 키워드 겹침까지 확장
  - 문서 상세와 문서 목록의 `새 문서 활용`이 현재 템플릿/소스 파일을 자동 연결하도록 정리
  - 검색 API의 문서 결과도 내부 공유 권한 기준으로 필터링해 검색/추천 흐름을 맞춤
  - `ai-context`, `design-prompt`, `diff`, `reflect` API에도 문서 접근/수정 권한 기준을 확장
  - `diff/analyze`, `quality-check`, `todo extract` 등 문서 파생 API에도 같은 권한 기준을 확장
  - 검색 화면에 역할/부서 기반 `검색 권한 범위` 안내를 추가
  - 대시보드 통계의 문서 수와 부서별 문서 분포를 접근 가능한 문서 기준으로 정리
  - 검색 화면에서 현재 역할, 기본 부서, 접근 가능한 부서 수, 현재 필터를 함께 보여주도록 권한 안내를 확장
  - 검색 결과가 `접근 가능한 범위 안에서만 계산된다`는 설명을 필터 UX 바로 아래에 연결
  - 대시보드 파일/문서 지표를 `접근 파일 / 접근 문서` 기준으로 통일
  - 대시보드 상단에 역할/기본 부서/조회 범위를 설명하는 권한 범위 카드를 추가

  - 배포 화면 기준으로 대시보드, 문서허브, 검색 화면의 밀도와 CTA 우선순위 실사용 검수
