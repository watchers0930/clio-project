# CLIO Document Detail IA

## 화면 목적
- 문서 상세는 단순 뷰어가 아니라 `공유`, `코멘트`, `반영`, `버전`, `후속 작업`이 모이는 핵심 운영 화면이다.

## 상단 영역
- 문서 제목
- 상태 배지
- 핵심 액션
  - 공유
  - 관련 검색
  - 새 버전 생성
  - 새 문서 활용
  - 다운로드

## 1차 본문 영역
- 문서 본문 뷰어
- 문서 메타 정보
  - 작성자
  - 생성/수정 시각
  - 템플릿
  - 소스 파일

## 운영 요약 영역
- `Document Ops` 요약
  - 코멘트 수
  - 활성 공유 링크 수
  - 버전 수
  - 내부 공유 수
- 최근 운영 타임라인
  - 최근 공유
  - 최근 코멘트
  - 최근 반영

## 협업 영역
- 공유 대상 패널
  - 외부 링크
  - 사용자 공유
  - 부서 공유
- 댓글 패널
  - 미반영
  - 보류
  - 반영
- 검토 요청과 메시지 연결

## 반영/이력 영역
- 버전 패널
  - 버전 목록
  - 각 버전에 반영된 코멘트 수
  - 일부 코멘트 요약
- 반영 상태
  - 누가
  - 언제
  - 무엇을 반영했는지

## 추천/후속 작업 영역
- 관련 문서
  - 같은 템플릿
  - 같은 소스
  - 제목 키워드 유사
- 후속 작업
  - 새 버전 생성
  - 새 문서 활용
  - 메모 연결
  - 관련 검색

## 우선순위 원칙
- `편집`보다 `공유`, `코멘트`, `반영`이 먼저 보여야 한다.
- 사용자는 상세 진입 후 다음 행동을 바로 이해해야 한다.
- 문서 상세는 읽기 완료 화면이 아니라 협업 진행 화면으로 설계한다.
