## 1. 맥락기술서 (Context Document)

### 1-1. 기능 목적

법무팀이 없는 중소 IT 기업은 계약서를 검토할 전문 인력이 없다.  
영업/PM 담당자가 직접 계약서를 읽고 서명하는 과정에서 불리한 조항, 누락된 필수 항목, 모호한 표현을 놓치는 리스크가 상존한다.

이 기능은 CLIO에 이미 구축된 파일 텍스트 추출 파이프라인(DOCX/HWPX/PDF)과 GPT-4o를 결합하여 계약서를 자동으로 분석하고, 리스크 수준별로 분류된 시각적 리포트를 제공한다.

**핵심 가치:**
- 법률 전문가 없이도 계약서 위험 항목 1차 검토 가능
- "을"의 입장에서 불리한 조항 자동 탐지 (IT 서비스 공급자 관점)
- 계약 체결 전 협상 포인트 명확화
- 한국 법률 관행 및 IT 업계 표준 계약 조항 기반 분석

### 1-2. 대상 사용자

| 사용자 | 사용 시나리오 |
|--------|-------------|
| 영업 담당자 | 고객사 계약서 초안 수령 후 체결 전 1차 검토 |
| PM | 시스템구축/유지보수 계약 조건 확인 |
| 대표/임원 | 고액 계약 최종 검토 보조 자료 |
| 법무 자문 | AI 분석 결과 기반으로 검토 포인트 집중 |

### 1-3. 기술 컨텍스트

| 구성요소 | 현황 |
|----------|------|
| 파일 텍스트 추출 | DOCX/HWPX/PDF 파이프라인 완성 (`lib/extractors/`) |
| 계약서 렌더러 | `contract-renderer.ts`, `contract-fields.ts` 기존 구현 |
| AI 엔진 | OpenAI GPT-4o 사용 중 |
| 언어/프레임워크 | TypeScript + Next.js 16 |
| 벡터 DB | pgvector (기존 RAG 파이프라인) |
| 인증/권한 | Supabase Auth + RBAC 완성 |

## 2. 분석 항목 정의

### 2-1. 리스크 분류 기준

| 수준 | 기호 | 정의 | 예시 |
|------|------|------|------|
| 상 (High) | 🔴 | 계약 체결 시 심각한 손실 또는 법적 분쟁 가능성 | 일방적 손해배상 무제한 책임, 지식재산권 전량 이전 강제 |
| 중 (Medium) | 🟡 | 불리하나 협상으로 조정 가능한 조항 | 과도한 지체상금률, 모호한 검수 기준 |
| 하 (Low) | 🟢 | 주의가 필요하나 일반적으로 수용 가능한 조항 | 단순 명기 누락, 표준보다 짧은 하자담보 기간 |

- AI 분석 설명 (왜 리스크인지)
- 권고사항 (어떻게 수정/협상할지)

### 6-3. DB 스키마 (신규)

```sql
CREATE TABLE contract_risk_analyses (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID REFERENCES users(id) ON DELETE CASCADE,
  file_name   TEXT NOT NULL,
  file_type   TEXT NOT NULL,               -- docx | hwpx | pdf
  contract_type TEXT NOT NULL,             -- system | maintenance | software | general
  perspective TEXT NOT NULL DEFAULT 'buyer_side', -- buyer_side | seller_side
  raw_text    TEXT,                        -- 추출된 원문 (보안 정책에 따라 암호화 검토)
  risk_result JSONB NOT NULL,              -- 분석 결과 JSON
  risk_count  JSONB NOT NULL,              -- { high: N, medium: N, low: N }
  status      TEXT NOT NULL DEFAULT 'pending', -- pending | processing | done | error
  created_at  TIMESTAMPTZ DEFAULT now(),
  updated_at  TIMESTAMPTZ DEFAULT now()
);
```

### 6-4. GPT-4o 프롬프트 전략

- 시스템 프롬프트: 한국 IT 계약 전문 법률 검토자 역할, 분석 항목 정의 주입
- 사용자 프롬프트: 계약서 원문 + 계약 유형 + 입장(갑/을) 전달
- 응답 형식: JSON 구조화 응답 (`response_format: { type: "json_object" }`)
- 토큰 제한 대응: 계약서가 긴 경우 청크 분할 분석 후 결과 병합

---

## 7. 우선순위 (MoSCoW)

### Must Have (P0 — 핵심, 반드시 구현)

- 파일 업로드 → 텍스트 추출 → AI 분석 → 결과 표시 기본 플로우
- 리스크 수준 분류 (상/중/하)
- 원문 발췌 + AI 설명 표시
- 면책 문구 표시
- 기본 분석 이력 저장

### Should Have (P1 — 중요, 이번 스프린트 내 구현)

- 계약서 유형 선택 (시스템구축/유지보수/소프트웨어/범용)
- 리스크 수준별 필터링
- 분석 진행 상태 UI (로딩/스트리밍)
- 리스크 요약 통계 헤더
- DOCX 리포트 다운로드
- 이력 목록 조회 페이지

### Could Have (P2 — 가능하면 구현)

- 갑/을 입장 선택 옵션
- 권고사항 구체화 (조항 수정 예시 제안)
- 분석 결과 인쇄 최적화

### Won't Have (P3 — 이번 버전 제외)

- 타사 법률 DB 연동
- 외부 공유 링크
- 영문 계약서 분석
- 실시간 협업 검토 (여러 명이 동시에 코멘트)

| 긴 계약서 처리 시간 초과 | 중 | 중 | 청크 병렬 분석, 30초 타임아웃 전 진행 상태 표시 |

## 9. 검증 기준 (Acceptance Criteria)

### 9-1. 기능 검증

| 검증 항목 | 방법 | 기대 결과 |
|-----------|------|-----------|
| DOCX 분석 | 실제 시스템구축계약서 DOCX 업로드 | 리스크 항목 목록 출력 |
| HWPX 분석 | HWPX 계약서 업로드 | 동일하게 분석 결과 출력 |
| PDF 분석 | 텍스트 기반 PDF 업로드 | 분석 결과 출력 |
| 스캔 PDF | 이미지 스캔 PDF 업로드 | "텍스트 추출 불가" 안내 |
| 리스크 분류 | 손해배상 무제한 조항이 포함된 계약서 | 해당 항목이 "상" 등급으로 탐지 |
| 누락 항목 탐지 | 검수 기준 없는 계약서 | B-03 항목 탐지 |
| 필터링 | "상" 필터 클릭 | 상위 리스크 항목만 표시 |
| 다운로드 | DOCX 다운로드 버튼 클릭 | 리포트 파일 정상 다운로드 |
| 이력 저장 | 분석 완료 후 이력 목록 조회 | 방금 분석한 건이 목록에 표시 |
| 면책 문구 | 결과 페이지 하단 | 면책 문구 표시 확인 |

### 9-2. 성능 검증

| 검증 항목 | 기준 |
|-----------|------|
| A4 10페이지 계약서 분석 완료 시간 | 30초 이내 |
| A4 30페이지 계약서 처리 가능 여부 | 청크 분할로 처리 완료 |
| 동시 2건 분석 요청 | 각각 독립적으로 정상 처리 |

### 9-3. 정확도 검증 (사전 샘플 테스트)

- 샘플 계약서 5건 (시스템구축 3건, 유지보수 2건) 수동 리뷰 결과와 AI 분석 결과 비교
- 주요 리스크 항목(상/중) 탐지율 목표: **80% 이상**
- 오탐(리스크 아닌 것을 리스크로 탐지) 허용 기준: **20% 이하**
