# CLIO Document Sharing Policy

## 목적
- 문서 공유 방식을 `링크`, `사용자`, `부서` 3축으로 정리한다.
- 공유 이후 흐름을 `검토 요청 -> 코멘트 -> 반영`까지 연결한다.

## 공유 방식

## 1. 링크 공유
- 외부 또는 비로그인 대상에게 문서를 전달할 때 사용한다.
- `shared_links` 기반으로 동작한다.
- 만료일, 비밀번호, 조회 수를 관리할 수 있다.
- 문서 배포 수단이지 내부 권한 자체를 의미하지는 않는다.

## 2. 사용자 공유
- 특정 개인에게 문서를 열람시키고 검토를 요청할 때 사용한다.
- `document_permissions.granted_to_user`로 관리한다.
- 가장 명확한 책임 기반 공유 방식이다.

## 3. 부서 공유
- 특정 부서 전체에 문서를 열람시키고 협업을 여는 방식이다.
- `document_permissions.granted_to_dept`로 관리한다.
- 팀 검토, 부서 간 협업, 표준안 전달에 적합하다.

## 현재 구현 기준
- 문서 공유 모달에서 아래 흐름을 지원한다.
  - 외부 링크
  - 사용자/부서 내부 공유
  - 검토 요청 메시지 발송
- 파일 공유도 같은 구조를 지원한다.
- 문서 상세에서는 내부 공유 수와 최근 공유 대상을 보여준다.
- 공유 문서 화면에서는
  - 나에게 공유된 문서
  - 내가 공유 중인 문서
  - 를 분리해 보여준다.

## 검토 요청 원칙
- 공유는 가능한 한 `검토 요청`과 같이 제공한다.
- 메시지 기능은 일반 대화보다 `문서 공유 + 검토 요청`에 우선 초점을 둔다.
- 공유 후에는 자연스럽게 `코멘트/검토` 화면으로 이어져야 한다.

## 부서 간 공유 운영 원칙
- 부서 간 공유는 예외가 아니라 공식 협업 수단으로 본다.
- 단, 문서 노출은 명시적 공유가 있을 때만 확장한다.
- 다른 부서 문서를 검색에서 보게 되는 것도 공유를 전제로 한다.

## 권장 운영 규칙
- 검토가 필요한 문서는 `사용자 공유`를 우선 사용한다.
- 참고용 전달은 `부서 공유`를 우선 사용한다.
- 외부 전달본은 `링크 공유`로 분리한다.
- 초안 단계에서도 공유는 가능하지만, 완료 문서 전환 이후에는 링크/대상 정리를 다시 확인한다.

## 후속 흐름
- 공유
- 코멘트 작성
- 댓글 반영
- 버전 확인
- 새 문서 활용

CLIO에서 공유는 전달의 끝이 아니라 문서 운영의 시작 단계다.
