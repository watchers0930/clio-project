# CLIO Core Scenarios

## 시나리오 1. 업로드 후 바로 검토 요청
- 사용자가 `문서허브`에서 파일을 업로드한다.
- 최근 업로드 패널에서 `공유` 또는 `검토 요청`을 선택한다.
- 사용자/부서/링크 기준으로 공유 대상을 정한다.
- 메시지 채널로 검토 요청을 보낸다.
- 수신자는 문서 상세나 공유 문서 화면에서 바로 코멘트를 남긴다.

## 시나리오 2. 공유 문서 검토와 반영
- 다른 부서 사용자가 `공유 문서`에서 공유받은 문서를 연다.
- 문서 상세의 댓글 패널에서 코멘트를 남기거나 보류 의견을 확인한다.
- 작성자는 `코멘트/검토` 화면에서 미반영 코멘트를 우선순위대로 본다.
- 작성자는 코멘트를 반영하고 새 버전을 만든다.
- 버전 패널에서 어떤 코멘트가 반영됐는지 확인한다.

## 시나리오 3. 검색에서 업무 재진입
- 사용자가 `AI 검색`에서 계약서, 보고서, 회의 문서를 찾는다.
- 검색 상단의 `바로 이어볼 문서 작업` 카드에서 가장 적절한 문서를 고른다.
- 결과 카드에서 `문서 열기`, `코멘트 보기`, `공유`, `새 문서 활용` 중 다음 행동을 선택한다.
- 관련 문서 추천과 최근 검색 흐름을 참고해 작업을 이어간다.

## 시나리오 4. 문서 재활용으로 보고서 작성
- 사용자가 기존 문서 상세나 보고서 허브에서 `새 문서 활용`을 누른다.
- 문서 생성 화면이 기존 템플릿과 소스 파일을 자동 연결한 상태로 열린다.
- 사용자는 보고서 초안을 수정하고 필요한 추가 문서를 검색으로 다시 붙인다.
- 완료 후 다시 공유와 코멘트 흐름으로 이어간다.

## 시나리오 5. 회의/메모에서 문서 작업으로 연결
- 사용자가 `회의` 허브에서 다가오는 회의를 연다.
- `회의록 작성`으로 문서 생성 흐름에 들어간다.
- 회의 중 남긴 메모는 문서 연결 메모로 축적한다.
- 회의 후 관련 문서 검색, 후속 문서 작성, 공유, 코멘트 반영으로 이어진다.

## 시나리오 구성 원칙
- 모든 시나리오는 `저장 -> 공유 -> 코멘트 -> 반영 -> 검색 -> 재활용` 중 일부 또는 전체를 지난다.
- 확장 기능은 항상 문서 흐름 위에 놓인다.
- 독립 기능처럼 보이는 화면도 최종 목적은 문서 운영 흐름 복귀다.
