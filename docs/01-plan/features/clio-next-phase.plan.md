## 1. 맥락기술서 (Context Document)

### 1-1. 프로젝트 개요

CLIO는 기업 내부용 RAG 기반 AI 문서관리 시스템이다.  
Next.js 16 + Supabase + OpenAI GPT-4o 스택으로 구축되었으며,  
파일 업로드 → 텍스트 추출 → 벡터 임베딩 → AI 문서 생성 전 파이프라인이 실운영 상태다.

### 1-2. 현재 구현 상태 요약

| 모듈 | 완성도 | 비고 |
|------|--------|------|
| 인증 (Auth) | ✅ 완성 | Supabase Auth + 쿠키 세션 |
| 파일 관리 | ✅ 완성 | 멀티파트 업로드, 50MB 제한, NFC 정규화 |
| AI 파이프라인 | ✅ 완성 | 텍스트 추출 → 청킹 → pgvector 임베딩 |
| 시맨틱 검색 | ✅ 완성 | pgvector + 텍스트 폴백 + AI 요약 |
| 멀티포맷 문서 생성 | ✅ 완성 | DOCX/HWPX/XLSX/PPTX/PDF 전체 지원 |
| 결재 워크플로우 | ✅ 완성 | 요청/승인/반려, 결재함 UI |
| 메시지 (채팅) | ✅ 완성 | DM/채널, 첨부파일, 5초 폴링 |
| 일정/할일 | ✅ 완성 | 월간 캘린더, CRUD |
| 설정 (부서/사용자) | ✅ 완성 | RBAC, 역할 관리 |
| STT 회의록 | ✅ 완성 | Whisper-1 + 자동 요약 |
| 계약서 입력폼 | ✅ 완성 | 시스템구축계약서 스키마, 직접 치환 렌더러 |

### 1-3. 현재 버전의 한계 및 기술 부채

분석을 통해 발견된 실제 문제들이다.

#### [버그] users.position 컬럼 미존재
- `src/app/api/generate/route.ts`에서 `userData?.position` 참조
- DB schema.sql 및 types.ts에 `position` 필드 없음
- 문서 생성 시 작성자 직급이 undefined로 처리됨
- **v3.5.5에서 position 컬럼 추가했다는 기록 있으나 마이그레이션 파일 없음**

#### [기술부채] 실시간 메시지 폴링 방식
- 메시지 채널을 10초 폴링으로 구현 (Supabase Realtime 미사용)
- 메시지 수신 지연 최대 10초, 서버 부하 지속 발생
- 사용자 수 증가 시 병목 포인트

#### [기술부채] localStorage 이중 관리
- `localStorage.getItem('clio_user')`가 다수 페이지에서 직접 호출
- Zustand `auth-store`와 상태 이중 관리 → 동기화 불일치 위험
- 세션 만료 시 stale 데이터 잔류 가능

#### [UX] 클라이언트 에러 피드백 부재
- 여러 페이지에서 `catch {}` 빈 블록으로 에러 무시
- 사용자가 API 실패를 인지하지 못하는 케이스 다수

#### [보안] .env.local.example 불완전
- `OPENAI_API_KEY`, `INTERNAL_API_SECRET` 예제 파일에 누락
- 신규 환경 세팅 시 AI 기능 전체 동작 불가

#### [미구현] 전자서명/도장
- 계약서 생성 후 서명/도장 삽입 기능 없음
- 계약서 완결성의 핵심 요소

#### [미구현] 계약서 템플릿 확장
- 시스템구축계약서 외 유지보수/소프트웨어 구축 등 스키마 없음

#### [미구현] AI 질의응답 강화
- 현재 벡터 검색은 파일 청크 기반
- 사용자가 "이 계약서의 납기일은?" 같은 자연어 질문을 문서 단위로 물을 수 없음

## 2. 후속 작업 계획서

### 우선순위 기준
- **P0**: 버그/데이터 무결성 — 즉시 수정
- **P1**: 핵심 UX/기능 완성 — 이번 스프린트
- **P2**: 기능 확장 — 다음 스프린트
- **P3**: 고도화 — 이후 검토

---

### P0 — 버그 수정 (즉시)

#### P0-1. users.position DB 마이그레이션
**문제:** generate API에서 `userData?.position` 참조하나 DB 컬럼 없음  
**해결:**
```sql
ALTER TABLE users ADD COLUMN IF NOT EXISTS position TEXT DEFAULT '';
```
- `supabase/migrations/006_users_position.sql` 작성
- `src/lib/supabase/types.ts`의 Users Row에 `position: string` 추가
- Supabase 대시보드 직접 실행 후 확인

**영향 범위:** `api/generate/route.ts`, `api/auth/me`, `api/users`

---

#### P0-2. .env.local.example 완성
**문제:** OPENAI_API_KEY, INTERNAL_API_SECRET 누락  
**해결:** `.env.local.example`에 두 변수 추가 (더미값으로)

---

### P1 — 핵심 UX 완성 (이번 스프린트)

#### P1-1. localStorage 단일화 (auth-store로 통합)
**문제:** `localStorage.getItem('clio_user')` 직접 호출 다수 존재  
**해결:**
- `useAuthStore()` 훅으로 전부 교체
- `auth-store.ts`에 `position` 필드 추가
- 로그인 시 Zustand persist에 position 포함

**영향 파일:** messages/page.tsx, documents/page.tsx, files/page.tsx 등 다수

---

#### P1-2. 클라이언트 에러 토스트 시스템
**문제:** catch{} 빈 블록으로 에러를 사용자에게 숨김  
**해결:**
- `components/ui/toast.tsx` 전역 토스트 컴포넌트 추가
- `app-layout.tsx`에 ToastProvider 포함
- 주요 페이지 catch 블록에 `showToast('오류 메시지')` 적용

**대상 페이지:** files, documents, templates, approvals

---

#### P1-3. 계약서 템플릿 확장
**문제:** 시스템구축계약서 외 타입 없음  
**해결:**
- `lib/contract-fields.ts`에 `유지보수계약서`, `소프트웨어구축계약서` 스키마 추가
- 각 스키마: 기본정보/금액/발주자/공급자 구조 동일, 특수 조항만 분기
- `isContractTemplate()` 판별 로직 확장

---

### P2 — 기능 확장 (다음 스프린트)

#### P2-1. 실시간 메시지 (Supabase Realtime 전환)
**현황:** 10초 폴링 → WebSocket으로 교체  
**해결:**
- `messages/page.tsx`에서 `supabase.channel()` + `.on('postgres_changes')` 구독
- 채널 이탈 시 `supabase.removeChannel()` 정리
- 폴링 코드(`setInterval`) 완전 제거

**기대효과:** 즉시 수신, 서버 요청 90% 감소

---

#### P2-2. 전자서명/도장 삽입
**목표:** DOCX/HWPX 생성 문서에 서명 이미지 삽입  
**구현 방향:**
- 설정 페이지에서 사용자별 서명 이미지 업로드 (PNG, 투명배경)
- `users` 테이블에 `signature_path` 컬럼 추가
- DOCX: `docx` 라이브러리의 `ImageRun`으로 서명 위치 삽입
- HWPX: XML `<hp:image>` 태그로 서명 이미지 삽입
- 결재 완료 시 자동 삽입 옵션

---

#### P2-3. AI 문서 질의응답 강화
**목표:** 문서/파일 기반 자연어 Q&A  
**현황:** 현재는 파일 청크 벡터 검색 후 AI 요약만 가능  
**구현 방향:**
- `/api/chat` 라우트 신설
- 선택한 파일 or 문서 범위 지정 → 청크 벡터 검색 → GPT-4o로 답변 생성
- `search/page.tsx`에 채팅 UI 패널 추가 (현재 검색 결과 우측)
- 대화 히스토리 유지 (messages 배열 상태 관리)

---

### P3 — 고도화 (이후 검토)

#### P3-1. 문서 버전 관리
- documents 테이블에 parent_id 컬럼으로 버전 체인
- 버전 히스토리 UI (타임라인 형태)

#### P3-2. 외부 공유 링크
- 문서/파일 외부 공유용 토큰 기반 링크 생성
- 만료일 설정, 비밀번호 보호 옵션

#### P3-3. 대시보드 고도화
- 파일 업로드 추이 차트 (주간/월간)
- 결재 처리 현황 차트
- 부서별 문서 생성량 통계
