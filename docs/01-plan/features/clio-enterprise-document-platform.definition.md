# CLIO Product Definition

## 한 줄 정의
- CLIO는 기업이 생산하고 관리하는 모든 문서를 저장, 공유, 검색, 검토, 업데이트하는 문서 운영 플랫폼이다.

## 제품의 중심 뼈대
- 저장
- 공유
- 코멘트
- 반영
- 검색
- 재활용

## 핵심 사용자 흐름 6단계

### 1. 저장
- 시작 화면은 `문서허브(/files)`다.
- 파일 업로드, 최근 업로드, 연결 문서 상태를 여기서 본다.
- 문서는 저장 즉시 `공유`, `검토 요청`, `관련 문서 연결`, `후속 문서 작성` 흐름으로 이어진다.

### 2. 공유
- 공유는 `문서 상세`, `검색 결과`, `파일 허브` 어디에서나 시작할 수 있다.
- 공유 방식은 `외부 링크`, `사용자`, `부서` 3축이다.
- 공유의 목적은 전달이 아니라 검토 대상을 여는 것이다.

### 3. 코멘트
- 공유받은 사용자는 문서 상세의 댓글 패널에서 코멘트를 남긴다.
- 운영자는 `공유 문서`, `코멘트/검토` 화면에서 미반영 의견을 모아서 본다.
- 메시지는 일반 대화보다 `문서 공유 + 검토 요청` 흐름에 가깝게 사용한다.

### 4. 반영
- 작성자나 관리자는 코멘트를 `미반영 / 보류 / 반영` 상태로 관리한다.
- 코멘트 반영은 버전 이력과 연결된다.
- 문서 상세는 읽기 화면이 아니라 `공유 상태`, `코멘트`, `버전`, `후속 작업`을 함께 보는 운영 화면이다.

### 5. 검색
- 검색은 단순 조회가 아니라 `업무 재진입점`이다.
- 검색 결과에서 바로 `문서 열기`, `공유`, `코멘트 보기`, `새 문서 활용`으로 이어진다.
- 검색은 접근 가능한 문서 범위와 부서 맥락을 함께 보여준다.

### 6. 재활용
- 확정된 문서는 `새 문서 활용`, `회의록`, `보고서`, `메모`, `계약 리스크`로 이어진다.
- 재활용은 기존 문서의 템플릿, 소스 파일, 버전 맥락을 최대한 이어받는다.
- CLIO의 확장 기능은 모두 이 마지막 단계 위에 올라간다.

## 제품이 아닌 것
- 계약 리스크 단일 제품이 아니다.
- 문서 생성 단일 툴이 아니다.
- 메신저/캘린더 중심 협업툴이 아니다.

## 제품 위에 올라가는 확장 기능
- 문서 생성
- 계약 리스크
- 회의록 작성
- 보고서 작성
- 메모

## 기본 원칙
- 문서는 보관 대상이 아니라 계속 운영되는 업무 자산이다.
- 부서마다 문서 유형은 달라도 공통 흐름은 같다.
- 플랫폼은 부서 업무를 통제하는 것이 아니라 공통 문서 흐름을 지원한다.

## 첫 화면에서 전달해야 하는 메시지
- 문서허브에서 시작한다.
- 저장된 문서를 공유하고 반영한다.
- 다시 검색하고 재활용한다.
- 전문 기능은 필요할 때 문서 맥락 위에서 사용한다.

## 화면별 역할 요약
- `대시보드`
  - 접근 가능한 문서 기준 운영 현황 요약
- `문서허브`
  - 저장과 운영 시작점
- `공유 문서`
  - 나에게 공유된 문서와 내가 공유 중인 문서 확인
- `코멘트/검토`
  - 미반영/보류 코멘트 큐 처리
- `문서 상세`
  - 공유, 코멘트, 반영, 버전의 핵심 운영 화면
- `AI 검색`
  - 검색 후 즉시 후속 작업으로 이동하는 재진입점
- `문서 생성`
  - 새 초안, 새 버전, 새 문서 활용의 실행 화면
