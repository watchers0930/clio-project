# CLIO Public Document Policy

## 목적
- `공용 문서`를 기본 공개 상태가 아니라 예외적으로 운영되는 배포 상태로 정의한다.
- 내부 협업 문서와 외부 배포 문서를 명확히 구분한다.

## 기본 원칙
- CLIO의 기본 문서 상태는 `작성자 소유 문서`다.
- 내부 협업은 `사용자 공유` 또는 `부서 공유`로 확장한다.
- `공용 문서`는 전사 공지, 표준 템플릿, 외부 배포본처럼 명시적으로 공개 의도가 있는 문서에만 사용한다.
- 아직 공용 문서 전용 권한 모델이 구현되지 않았다면, 운영상으로는 `공유 링크` 또는 `공유 문서` 흐름으로 대체한다.

## 공용 문서로 볼 수 있는 대상
- 전사 공지 문서
- 표준 계약서/제안서/보고서 샘플
- 부서 공통 운영 가이드
- 외부 고객 전달용 최종본

## 공용 문서가 아닌 대상
- 초안 문서
- 특정 프로젝트 내부 검토본
- 특정 사용자만 확인해야 하는 승인 전 문서
- 코멘트 반영이 진행 중인 작업 문서

## 운영 규칙
- 공용 전환 전에는 아래를 확인한다.
  - 최신 버전인지
  - 외부 공개 가능한 내용인지
  - 내부 코멘트가 정리되었는지
  - 대체 문서나 만료 계획이 필요한지
- 공용 문서는 `링크 배포`와 `내부 참조`를 분리해 관리한다.
- 공용 문서라도 수정 권한은 자동 공개하지 않는다.
- 공용 문서의 원본 작업은 여전히 작성자 또는 관리자 관리 아래 둔다.

## 현재 제품 구조에서의 해석
- 현재 CLIO는 `전사 공개 기본값`보다 `명시적 공유 기본값`에 가깝다.
- 그래서 공용 문서 정책은 `누구나 보이는 문서`를 늘리는 정책이 아니라 `공식 배포 문서를 엄격히 분리하는 정책`으로 이해해야 한다.
- 실제 구현 전까지는 아래 조합으로 운영한다.
  - 내부 공용: 부서 공유 또는 다수 사용자 공유
  - 외부 공용: 만료일과 비밀번호가 있는 공유 링크

## 향후 구현 권장
- `공용 문서` 상태 필드 추가
- 공용 문서 전용 목록 화면 추가
- 공용 전환 승인 로그 기록
- 공용 문서 만료/교체 정책 추가

## 운영 해석
- CLIO에서 공용 문서는 기본값이 아니라 `공식 배포 상태`다.
- 핵심 흐름은 여전히 `저장 -> 공유 -> 코멘트 -> 반영 -> 검색 -> 재활용`이며, 공용 문서는 그 결과물 중 일부만 해당한다.
