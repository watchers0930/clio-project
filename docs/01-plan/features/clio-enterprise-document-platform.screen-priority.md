# CLIO Screen Priority

## P0 핵심 운영 화면
- 문서허브
  - 저장 이후 첫 행동이 결정되는 시작점
- 문서 상세
  - 공유, 코멘트, 반영, 버전의 중심 화면
- AI 검색
  - 검색 이후 업무 재진입점

## P1 협업 화면
- 공유 문서
  - 나에게 공유된 문서와 내가 공유 중인 문서 확인
- 코멘트/검토
  - 미반영 코멘트 처리 큐
- 메시지
  - 공유와 검토 요청 전달

## P2 문서 확장 화면
- 문서 생성
  - 새 초안, 새 버전, 재활용 실행
- 회의
  - 회의 문서 시작점
- 보고서
  - 보고 문서 작성 흐름
- 메모
  - 문서 연결 메모

## P3 운영/정책 화면
- 사용자
  - 역할과 부서 배치 관리
- 부서
  - 조직 구조 관리
- 템플릿
  - 전사 공용/부서 전용 양식 관리
- 서명/설정
  - 서명과 운영 정책 확인

## 우선순위 해석
- CLIO의 첫 인상은 `문서허브`, `문서 상세`, `검색`이 결정한다.
- 협업 화면은 핵심 운영 흐름을 끊지 않도록 중간 단계 진입면으로 둔다.
- 확장 기능은 존재감보다 연결성을 우선 평가한다.
- 관리 화면은 운영 정책을 뒷받침하지만 제품 정체성을 앞서지 않는다.
