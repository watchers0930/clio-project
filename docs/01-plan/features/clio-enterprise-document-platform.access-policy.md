# CLIO Document Access Policy

## 목적
- 문서 접근 기준을 `작성자`, `공유 대상`, `관리자` 중심으로 통일한다.
- 검색, 상세, 다운로드, AI 보조 기능까지 같은 기준으로 동작하게 한다.

## 기본 원칙
- 문서는 작성자가 기본 소유자다.
- 문서는 사용자 또는 부서 단위로 내부 공유할 수 있다.
- 외부 공유 링크는 배포 수단이며 내부 권한을 대체하지 않는다.
- 관리자(`admin`)는 모든 문서에 접근할 수 있다.

## 접근 대상
- `작성자`
  - 항상 본인 문서에 접근할 수 있다.
- `내부 공유된 사용자`
  - `document_permissions.granted_to_user`로 직접 부여된 경우 접근할 수 있다.
- `내부 공유된 부서 소속 사용자`
  - `document_permissions.granted_to_dept`로 부여된 부서에 속하면 접근할 수 있다.
- `관리자`
  - 전체 문서에 접근하고 관리할 수 있다.

## 현재 구현 기준
- 문서 목록
  - 본인 문서 + 내부 공유된 문서만 노출
- 문서 상세
  - 접근 가능한 문서만 조회 가능
- 문서 다운로드
  - 접근 가능한 문서만 다운로드 가능
- 댓글 조회/작성
  - 접근 가능한 문서에서 가능
- 버전 조회
  - 접근 가능한 문서에서 가능
- 검색 결과
  - 접근 가능한 문서만 검색 결과에 포함
- AI/파생 API
  - `ai-context`
  - `design-prompt`
  - `diff`
  - `diff/analyze`
  - `quality-check`
  - `todo extract`
  - 모두 접근 가능한 문서 기준으로 제한

## 수정/관리 기준
- `작성자`
  - 문서 수정
  - 문서 삭제
  - 댓글 반영
  - 새 버전 생성
- `관리자`
  - 전체 문서 관리 가능
- `공유받은 사용자/부서`
  - 열람
  - 코멘트 작성
  - 후속 검토
  - 직접 수정이나 삭제는 불가

## 검색/대시보드 반영 원칙
- 검색 결과 문서는 접근 가능한 문서만 보여준다.
- 관련 문서 추천도 접근 가능한 문서만 포함한다.
- 대시보드의 문서 수와 부서별 문서 분포도 접근 가능한 문서 기준으로 집계한다.

## 운영 해석
- CLIO의 문서 접근 정책은 `전사 공개`가 기본이 아니다.
- 기본값은 `작성자 소유`이며, 협업은 `공유`로 확장한다.
- 그래서 검색, 공유 문서, 코멘트/검토 흐름은 모두 `공유된 문서 운영` 구조 위에 놓인다.
