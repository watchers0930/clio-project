## 작업 배경

현재 CLIO는 기능이 부족한 것이 아니라 기능이 분산되어 보이는 것이 문제다.

핵심 개선 목표는 아래 3가지다.

1. CLIO가 `문서 중심 AI 업무 플랫폼`이라는 점을 UI에서 명확히 보이게 만들 것
2. 파일 업로드 이후 사용자가 해야 할 다음 행동을 시스템이 안내하게 만들 것
3. `파일 관리 -> AI 검색 -> 문서 생성 -> 계약 리스크`가 하나의 흐름처럼 이어지게 만들 것

## 이번 작업 범위

이번 작업은 `Phase 1`만 수행한다.

즉, 아래 범위만 구현한다.

- 사이드바 메뉴 우선순위와 정보구조 정리
- 대시보드 개편
- 대시보드 CTA 및 위젯 재배치
- 문서 중심 제품 메시지가 드러나도록 첫 화면 개선

이번 작업에서 하지 말 것:

- 파일 관리 화면 대규모 개편
- 검색 화면 기능 확장
- 계약 리스크 API 구조 변경
- 메시지/일정/메모 기능 추가
- DB 마이그레이션

## 구체적인 구현 목표

### 1. 사이드바

`src/components/layout/sidebar.tsx`를 검토하고 아래 방향으로 개선한다.

- 문서 관련 메뉴가 중심으로 보이게 정렬한다.
- 시각적으로도 메인 기능과 보조 기능의 우선순위가 느껴지게 만든다.
- 현재 메뉴 자체를 크게 삭제하지는 않되, 그룹감이나 흐름이 보이도록 개선한다.
- 가능한 범위에서 라벨/배치/스타일로 제품 중심축을 드러낸다.

### 2. 대시보드

`src/app/(app)/dashboard/page.tsx`를 검토하고 아래 방향으로 개편한다.

- 첫 화면에서 CLIO가 문서 업무 플랫폼이라는 점이 보여야 한다.
- 빠른 액션은 아래 우선순위로 정렬한다.
  - 파일 업로드
  - AI 검색
  - 문서 생성
  - 계약 리스크
- 최근 파일, 최근 문서, 최근 활동이 끊어진 정보처럼 보이지 않게 정리한다.
- 가능하면 사용자가 “다음에 무엇을 해야 하는가”를 바로 이해할 수 있게 만든다.
- 대시보드가 단순 통계판이 아니라 작업 출발점처럼 보이게 만든다.

### 3. 제품 메시지

대시보드의 상단 영역 또는 핵심 카드 영역에 아래 메시지가 자연스럽게 드러나야 한다.

- CLIO는 문서를 업로드하고
- 찾고
- 생성하고
- 검토하고
- 계약 리스크까지 확인하는 플랫폼이라는 점

직접 문구를 복붙할 필요는 없지만, 이 가치 구조가 UI에서 읽혀야 한다.

## 구현 원칙

반드시 아래 원칙을 지킨다.

1. 기존 기능을 망가뜨리지 않는 선에서 UI/IA를 개선한다.
2. 새 컴포넌트가 필요하면 작게 분리하되 과도한 추상화는 피한다.
3. 기존 스타일 시스템과 톤을 최대한 존중하되, 우선순위는 더 선명하게 만든다.
4. 보여주기용 문구보다 실제 행동 유도가 중요하다.
5. 코드 변경은 최소 범위가 아니라, 목표를 달성하는 데 필요한 충분한 범위로 한다.

## 작업 절차

아래 순서로 진행한다.

1. 관련 파일을 읽고 현재 구조를 요약한다.
2. 구현 계획을 짧게 세운다.
3. 사이드바부터 수정한다.
4. 대시보드를 수정한다.
5. 필요하면 작은 공용 컴포넌트/상수를 정리한다.
6. 실행 가능한 검증을 수행한다.
7. 최종적으로 변경 내용과 검증 결과를 요약한다.

## 산출물 요구사항

최종 응답에는 반드시 아래 내용을 포함한다.

1. 무엇을 왜 바꿨는지
2. 어떤 파일을 수정했는지
3. 사용자가 체감하는 변화가 무엇인지
4. 아직 남은 다음 단계가 무엇인지

## 검증 요구사항

가능하면 아래를 수행한다.

- 빌드 또는 타입체크
- 최소한 변경된 화면의 코드 레벨 검토
- 명백한 회귀 가능성 점검

테스트를 못 돌리면 이유를 명확히 적는다.

## 완료 기준

아래 질문에 예라고 답할 수 있어야 한다.

1. 대시보드 첫 화면에서 CLIO가 문서 중심 제품으로 보이는가?
2. 사용자가 첫 화면에서 바로 핵심 작업으로 진입할 수 있는가?
3. 사이드바가 기능 나열이 아니라 제품 구조처럼 보이는가?
4. 보조 기능이 메인 가치보다 앞에 서지 않는가?

작업을 시작하라.
