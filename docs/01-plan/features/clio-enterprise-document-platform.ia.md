# CLIO Information Architecture

## 1차 메뉴
- 문서 운영
  - 대시보드
  - 문서허브
  - 공유 문서
  - AI 검색
  - 문서 생성
- 협업
  - 메시지
  - 코멘트/검토
  - 회의
  - 일정/할일
  - 메모
- 업무 지원
  - 계약 리스크
  - 보고서
- 관리
  - 사용자
  - 부서
  - 템플릿
  - 서명/설정

## 메뉴 구조도

### 문서 운영
- `대시보드`
  - 접근 문서 현황
  - 부서별 문서 분포
- `문서허브`
  - 최근 업로드
  - 공유 대기
  - 코멘트 미반영
  - 최근 업데이트
- `공유 문서`
  - 나에게 공유된 문서
  - 내가 공유 중인 문서
- `AI 검색`
  - 검색 결과
  - 관련 문서 추천
  - 최근 검색 흐름
- `문서 생성`
  - 새 문서 작성
  - 새 버전 생성
  - 기존 문서 재활용

### 협업
- `메시지`
  - 공유 알림
  - 검토 요청
- `코멘트/검토`
  - 미반영 코멘트
  - 보류 코멘트
  - 검토 대상 문서
- `회의`
  - 다가오는 회의
  - 최근 회의 문서
- `일정/할일`
  - 일정 기준 문서 작업 연결
- `메모`
  - 문서 연결 메모
  - 메모에서 문서 재진입

### 업무 지원
- `계약 리스크`
  - 특정 문서 분석
  - 수정 제안
- `보고서`
  - 보고서 템플릿
  - 업데이트 보고서 작성

### 관리
- `사용자`
  - 사용자 역할
  - 부서 배치
- `부서`
  - 부서 구조
  - 운영 현황
- `템플릿`
  - 전사 공용 템플릿
  - 부서 전용 템플릿
- `서명/설정`
  - 서명 등록
  - 정책 확인
  - 운영 제어판

## 구조 원칙
- 문서 운영 메뉴가 항상 최상단에 온다.
- 협업 기능은 문서 흐름을 보조하는 위치에 둔다.
- 계약 리스크는 전문 기능이므로 문서 운영 아래가 아니라 보조 축에 둔다.
- 공유 문서와 코멘트/검토는 문서 운영 흐름의 중간 단계를 별도 진입면으로 보여준다.
- 회의, 보고서, 메모는 독립 툴처럼 보이더라도 실제 행동은 문서 생성과 재활용 흐름으로 이어진다.
- 관리 메뉴는 단순 설정 모음이 아니라 `사용자`, `부서`, `템플릿`, `정책`을 운영하는 제어판 구조로 둔다.

## 문서허브 역할
- 문서 저장소
- 공유 시작점
- 코멘트 반영 출발점
- 검색/생성/전문 분석 연결점

## 문서 상세 역할
- 본문 확인
- 공유 상태 확인
- 코멘트 확인
- 반영 상태 확인
- 버전 이력 확인
- 후속 작업 연결

## 추가 진입면 역할
- 공유 문서
  - 나에게 공유된 문서 진입
  - 내가 공유 중인 문서 점검
- 코멘트/검토
  - 미반영/보류 코멘트 큐 확인
  - 댓글 반영 우선순위 관리
- 회의
  - 회의 일정에서 회의록 작성으로 연결
  - 회의 문서에서 후속 보고 문서로 확장
- 보고서
  - 보고서 템플릿 선택
  - 관련 문서 검색
  - 업데이트 보고서 작성
- 메모
  - 문서에서 임시 메모 작성
  - 메모에서 검색/문서 생성으로 재진입
